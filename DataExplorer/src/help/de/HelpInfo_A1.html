Geräteeigenschafteneditor

Über das Gerätemenü oder als eigenständige Anwendung aufgerufen lassen sich neue oder vorhandene Geräteeigenschaftendateien erstellen oder anpassen.

Über den Knopf in der Kopfzeile kann eine Gerätedatei direkt aus dem Gerätedateienordner (Devices) geöffnet werden. Ist das der Fall, wird die Gerätedatei des aktuellen Gerätes geladen und ein Ändern der Datei ist nicht möglich. Als eigene Anwendung gestartet kann der Name, falls bekannt kann auch direkt eingegeben werden. Ist der Name der Datei nicht im Geräteordner enthalten, geht die Anwendung davon aus das eine neue Gerätedatei angelegt werden soll und fragt danach.








Hierbei wird dann eine Beispieldatei extrahiert, die dann angepasst werden kann. Durch Selektion des Knopfes auf der rechten Seite, kann über einen Dateiauswahldialog die Gerätedatei geöffnet werden.

Der Gerät Tabulator zeigt wie man Name und Herstellerangaben einträt.

Schaltet man die Implementierung aktiv, muss hier die Geräteklasse als Klassenname oder mit package, als voll qualifiziert, eingetragen werden. Eine Implementierung kann immer dann verwendet werden, wenn ein Gerät schon implementiert ist, nur der Name nicht passt. Als Beispiel soll hier der CSV2SerialAdapter genannt werden, Unterschiede bestehen nur in der Spezifikation des Datentextblocks. Als weiteres Beispiel können die vielen Kopien der Bantam Ladegeräte herangezogen werden, die sich im Aussehen und Namen unterscheiden. Hier möchete man die Daten doch unter seinem Gerätenamen abgelegt vorfinden.
Gibt man das 'package' nicht an beginnt das 'package' immer mit "osde.device.", gefolgt vom namen des Herstellers in Kleinbuchstaben umgewandelt und Leerzeichen, sowie Bindestriche entfernt. Aus dem Ger$Auml;tenamen wird der Klassenname ebenfalls durch entfernen von Leerzeichen und Bindestrichen berechnet.

Der Bildname kann von Hand eingegeben werden, falls auf ein, im Geräte Plug-in, vorhandenes Bild in der Geräteauswahl angezeigt werden soll. Wird über den Knopf auf der rechten Seite eine Bilddatei ausgewählt, wird diese sofort auf die erforderliche Größe (225x165) skaliert. Hierbei wird auch angeboten, die Bilddatei sofort in das Geräte Plug-in einzupacken. Dies kann aber nur in einem temporären Ordner geschehen, da nur hier Schreibrechte garantiert sind. Gegebenenfalls muss später das Plug-in in den Anwendungsordner kopiert werden.
Genaue Anweisungen, wo sich das modifizierte Plug-in befindet und wo es auszutauschen ist, werden als Messagebox angezeigt.

Hinweis : Diese Änderung geht bei einem Anwendungsupdate verloren, wenn die Änderungen nicht allgemeingültig und der Anwendung beim bauen zur Verfügung gestellt wird. Gegebenenfalls selbst für eine Sicherung sorgen.

Für den seriellen Port sind die entsprechenden konfigurationsparameter einzutragen. Da hier alle Auswahlmöglichkeiten über Auswahldialoge eingestellt werden sind weiter Hinweise nicht nötig.

Hinweis : Werden Timeoutzeiten angegeben muss die Implementierung so geschrieben sein, dass diese auch benutzt werden. Werden hier Werte eingetragen werden diese also nicht automatisch benutzt.
Die Beschreibung der seriellen Schnittstelle ist optional, weil es Greäte gibt, die einen virtuellen seriellen Input über Textdaten erhalten. Beispielsweise haben Geräte die das CSV2SerialAdapter Plug-in benutzen keine serielle Schnittstelle.




















Zur Zeitbasis konfiguration ist einfach.
Ein Wert grösser 0 Millisekunden als Zeitabstand zwischen den Messpunkten wird als konstanter Zeitabstand angenommen. Bei einem Zeitwert kleiner 0 Millisekunden, wird davon ausgegengen, dass das Gerät die Zeit vorgibt. Das muss nicht heissen, dass der Abstand zwischen den Messpunkten Konstant ist. Vorrangig muss hier die Geräteimplementierung das Richtige daraus machen.

Hinweis : Das UniLog als Beispiel hat einen einstellbaren aber konstanten Zeitabstand zwischen den Messpunkten. Schlussendlich muss auch hier die Geräteimplementierung für das richtige Verhalten sorgen.




















Hier werden 2 verschiedene Datanblockkonfigurationen gegenübergestellt. Die UniLog Konfiguration beschreibt binäre Daten der konstanten Größe von 24 Byte. Sollte die maximale Anzahl von Datenbytes 24 betragen, können als auch mal nur z.B. 20 Bytes zur Verfügung stehen, müsste hier -24 eingetragen werden. Dadurch, dass als Datentyp BINARY gewählt ist können auch keine Trennzeichen eingegeben werden.

























Ist als Datentype TEXT gewählt müssen zusätzliche Konfigurationsdaten angegeben werden, die zum verwerten der Daten benötigt werden. Interessant sind auch die Dateiendung, die die Dateisuche einschränkt, sowie der Dateipfad, der ein Standardverzeichnis spezifiziert, wo die Daten erwartet werden. Beides sin optionale Angaben, helfen aber, wenn Daten, wie bei dem DataVario von WStech auf einer Speicherkarte angeliefert werden, die beim einstecken in den Computer einen bestimmten Laufwerkspfad annimmt.

Hinweis : Auch diese Konfiguration ist optional, die Geräteimplementierung selbst könnte das alles behandeln.
